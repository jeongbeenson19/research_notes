## ✅ 1. **그리디 알고리즘이란?**

> **매 순간, 지금 상황에서 가장 좋아 보이는(가장 이득이 되는) 선택을 하는 알고리즘**  
> → "지금 이 순간 최적이면, 결국 전체도 최적일 것이다"라는 직관에 기반

즉, 전체적인 최적해(global optimum)를 보장하는 것이 아니라, **매 순간의 국소 최적해(local optimum)** 를 반복적으로 선택하는 방식입니다.

---

## ✅ 2. **그리디 알고리즘의 기본 아이디어**

1. **해 선택 (Choice Property)**  
    → 현재 상황에서 가장 좋은 선택을 하나 고릅니다.
    
2. **적합성 검사 (Feasibility Check)**  
    → 지금 선택이 문제의 조건을 위반하지 않는지 확인합니다.
    
3. **해의 완성 (Solution Optimality)**  
    → 이러한 선택을 반복하며 전체 문제의 해답을 구성합니다.
    

---

## ✅ 3. **그리디 알고리즘이 적용 가능한 조건**

그리디 알고리즘이 **항상 최적해를 보장하려면** 다음 두 조건이 성립해야 합니다.

|조건|설명|
|---|---|
|**Greedy-choice property (탐욕적 선택 속성)**|**현재 순간의 최적 선택이 전체 문제의 최적해로 이어질 수 있어야 함**|
|**Optimal substructure (최적 부분 구조)**|문제를 부분 문제로 나누었을 때, 부분 문제의 최적해들이 모여 전체의 최적해가 되어야 함|

이 두 가지 조건이 만족되지 않는 문제에서는 **동적 계획법(DP)** 이나 **백트래킹**을 사용해야 합니다.

---

## ✅ 4. **그리디 알고리즘의 장단점**

### ✔ 장점

- 구현이 간단하고 직관적
    
- 시간 복잡도가 낮은 경우가 많음 (보통 `O(N log N)` ~ `O(N)` 수준)
    
- 큰 입력 크기에서도 빠른 근사 해를 구할 수 있음
    

### ✘ 단점

- 항상 최적해를 보장하지 않음
    
- 모든 문제에 적용 불가

## ✅ 5. **대표적인 그리디 알고리즘 문제**

|문제 유형|예시|간단 설명|
|---|---|---|
|**거스름돈 문제**|백준 11047 (동전 0)|가장 큰 동전부터 사용|
|**활동 선택 문제**|회의실 배정 문제(백준 1931)|종료 시간이 가장 빠른 회의부터 배정|
|**최소 대기시간**|ATM (백준 11399)|시간이 짧은 사람부터 정렬 후 계산|
|**최소 이동 연산**|회전하는 큐 (백준 1021)|매 순간 왼쪽, 오른쪽 중 더 적은 회전 선택|
|**허프만 코딩**|텍스트 압축|빈도가 낮은 문자끼리 먼저 묶기|

## ✅ 6. **예시 1 – 거스름돈 문제**

### 문제: 10원, 50원, 100원, 500원 동전으로 760원을 거슬러줄 때 최소 동전 개수?

### 그리디 알고리즘 적용

1. 가장 큰 단위부터 사용:
    
    - 500원 × 1개 → 남은 금액: 260원
        
    - 100원 × 2개 → 남은 금액: 60원
        
    - 50원 × 1개 → 남은 금액: 10원
        
    - 10원 × 1개 → 남은 금액: 0원
        
2. 총 **5개**로 완료
    

✔ **탐욕적 선택 속성과 최적 부분 구조를 만족**하므로 정답 보장

---

## ✅ 7. **예시 2 – 그리디가 실패하는 경우**

### 문제: 1원, 3원, 4원 동전으로 6원을 거슬러줄 때 최소 개수?

- **그리디 선택**: 4 → 1 → 1 (총 3개)
    
- **최적해**: 3 → 3 (총 2개)
    

→ 이 경우 **그리디는 정답 보장 X**, 대신 **DP**를 써야 함.

---

## ✅ 8. **그리디 vs 다이나믹 프로그래밍 비교**

|항목|그리디|DP|
|---|---|---|
|**선택 방식**|현재 최적 선택만 고려|모든 경우의 수를 고려|
|**최적해 보장**|일부 문제만 가능|대부분 가능|
|**시간 복잡도**|빠름|상대적으로 느림|
|**적용 문제**|거스름돈, 회의실 배정 등|배낭 문제, 최장 증가 부분 수열(LIS) 등|

## ✅ 9. **의사 코드 (그리디 알고리즘 일반 형태)**

```python
result = 0
while 문제의 해가 완성되지 않음:
    최적의 선택 = 현재 상황에서 가장 좋아 보이는 선택
    if 선택이 문제 조건에 부합:
        결과에 선택 반영
        result += 선택의 비용
    else:
        선택 폐기
print(result)
```

## ✨ 정리

- **그리디 알고리즘은 "현재 순간 최적"을 반복해서 빠르고 단순하게 푸는 방법**
    
- 단, **모든 문제에서 정답을 보장하지 않음** → **탐욕적 선택 속성**과 **최적 부분 구조**가 만족되어야 함
    
- 적용 가능한 전형적인 문제 유형이 정해져 있음
