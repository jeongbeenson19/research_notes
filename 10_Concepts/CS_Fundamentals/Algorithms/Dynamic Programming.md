
## ✅ **1. 동적 프로그래밍이란?**

동적 프로그래밍(DP)은 **복잡한 문제를 작은 부분 문제(Subproblem)로 나누어 해결한 뒤, 그 결과를 재활용하여 전체 문제를 해결하는 알고리즘 기법**입니다.

즉, 한 번 계산한 부분 문제의 결과를 **저장**해두고, **같은 계산을 반복하지 않고** 이미 구한 결과를 이용하는 방식입니다.

---

## ✅ **2. DP의 특징**

1. **부분 문제 반복 (Overlapping Subproblems)**
    
    - 큰 문제를 해결하는 과정에서 **같은 작은 문제를 여러 번** 해결해야 하는 상황이 발생합니다.
        
    - 예: 피보나치 수열 → `f(5)`를 구할 때 `f(3)`은 여러 번 계산됩니다.
        
2. **최적 부분 구조 (Optimal Substructure)**
    
    - 문제의 최적해가 **부분 문제의 최적해로 구성**될 수 있어야 합니다.
        
    - 예: `f(n) = f(n-1) + f(n-2)`와 같은 식으로 작은 문제의 해를 조합해서 큰 문제의 해를 구할 수 있어야 합니다.
        

---

## ✅ **3. DP를 적용하는 두 가지 방식**

### ① **탑다운 방식 (Top-Down, 재귀 + 메모이제이션)**

- **큰 문제를 먼저 호출**하고, 필요한 경우 **작은 문제를 재귀적으로 호출**하면서 해결
    
- 이미 계산된 부분 문제는 **메모(저장)** 하여 다시 계산하지 않음 → **메모이제이션(Memoization)**
    

예) 피보나치 수열

```python
dp = [0] * 100

def fib(n):
    if n == 0 or n == 1:
        return n
    if dp[n] != 0:
        return dp[n]
    dp[n] = fib(n-1) + fib(n-2)
    return dp[n]

print(fib(10))  # 55
```

---

### ② **바텀업 방식 (Bottom-Up, 반복문)**

- 작은 문제부터 차근차근 풀어가며 **점화식**을 이용해 큰 문제를 해결
    
- 배열이나 리스트에 값을 순차적으로 저장
    

예) 피보나치 수열

```python
dp = [0] * 100
dp[0] = 0
dp[1] = 1

for i in range(2, 100):
    dp[i] = dp[i-1] + dp[i-2]

print(dp[10])  # 55
```

---

## ✅ **4. DP의 핵심: 점화식 (Recurrence Relation)**

DP에서 가장 중요한 것은 **점화식**을 세우는 것입니다.

예를 들어, 앞서 주신 **1, 2, 3 더하기 문제**의 점화식은:

dp[n]=dp[n−1]+dp[n−2]+dp[n−3]dp[n] = dp[n-1] + dp[n-2] + dp[n-3]

초기값을 세팅한 후 이 점화식을 이용하면, 반복적으로 쉽게 계산할 수 있습니다.

---

## ✅ **5. DP와 단순 재귀의 차이**

|구분|단순 재귀|동적 프로그래밍 (DP)|
|---|---|---|
|**계산 중복**|동일한 부분 문제를 여러 번 계산|이미 계산한 결과를 재사용|
|**시간 복잡도**|지수 시간(O(2^n)) 가능성 높음|다항 시간(O(n))|
|**메모리 사용**|메모 사용 X (필요 없음)|메모리(배열/딕셔너리) 사용|
|**예시**|단순한 피보나치 구현|DP 피보나치 구현|

---

## ✅ **6. 정리**

- **DP = 큰 문제를 작은 문제로 나누고, 한 번 푼 작은 문제의 답을 저장 후 재활용하는 방식**
    
- 핵심은 **점화식 세우기 + 메모이제이션 or 테이블 저장**
