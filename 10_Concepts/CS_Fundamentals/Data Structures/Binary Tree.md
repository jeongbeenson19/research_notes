
## 🌳 이진 트리란?

- **정의**: 각 노드가 **최대 두 개의 자식 노드**를 가지는 트리
    
- 자식 노드는 보통 **왼쪽 자식(left child)** 과 **오른쪽 자식(right child)** 으로 구분됨
    
- **계층 구조**를 가지며, 루트(root)에서 시작해 하위 노드로 내려가는 구조
    

---

## ✅ 주요 용어

|용어|의미|
|---|---|
|루트(Root)|트리의 시작 노드|
|리프(Leaf)|자식이 없는 노드 (끝 노드)|
|내부 노드|최소 하나 이상의 자식을 가진 노드|
|서브트리|어떤 노드를 루트로 하는 하위 트리|
|깊이(Depth)|루트에서 해당 노드까지의 간선 수|
|높이(Height)|노드에서 가장 깊은 리프까지의 간선 수|

---

## ✅ 이진 트리 종류

1. **포화 이진 트리 (Full Binary Tree)**  
    모든 노드가 **0개 또는 2개의 자식 노드**만 가지는 트리
    
2. **완전 이진 트리 (Complete Binary Tree)**  
    마지막 레벨을 제외하고 **모든 레벨이 꽉 차 있고**, 마지막 레벨은 **왼쪽부터 채워지는 트리**
    
3. **정 이진 트리 (Proper Binary Tree)**  
    자식이 **0개 또는 2개**인 노드만 존재
    
4. **이진 탐색 트리 (Binary Search Tree, BST)**  
    왼쪽 서브트리는 **작은 값**, 오른쪽 서브트리는 **큰 값**을 가지도록 정렬된 이진 트리
    

---

## ✅ 예시 그림

```
       1
      / \
     2   3
    / \
   4   5
```

- 루트: 1
    
- 리프: 3, 4, 5
    
- 2는 자식 노드가 2개 있으므로 내부 노드
    

---

## ✅ 활용 예시

- **이진 탐색 트리 (BST)**: 정렬 및 검색 최적화
    
- **힙([[Heap]])**: 우선순위 큐 구현
    
- **허프만 트리**: 데이터 압축
    
- **이진 표현 파싱**: 수식 트리, 문법 트리

필요하다면 Python으로 이진 트리 구현 코드도 제공해줄 수 있어요. 더 알고 싶은 부분이 있을까요?

이진트리의 **Inorder(중위순회)**, **Preorder(전위순회)**, **Postorder(후위순회)** 는 노드를 방문하는 **트리 순회(Tree Traversal)** 방법 중 대표적인 세 가지입니다. 각각 **재귀적으로** 왼쪽 자식, 현재 노드, 오른쪽 자식을 방문하는 순서가 다릅니다.

---

### ✅ 1. **Inorder Traversal (중위 순회)**

**방문 순서:** `왼쪽 → 루트 → 오른쪽`

```plaintext
왼쪽 서브트리 순회  
→ 현재 노드 방문  
→ 오른쪽 서브트리 순회
```

- 이진 탐색 트리(BST)에서는 **오름차순 정렬된 값**을 얻을 수 있음.
    

#### 예시:

```
      A
     / \
    B   C
```

→ **Inorder:** `B A C`

---

### ✅ 2. **Preorder Traversal (전위 순회)**

**방문 순서:** `루트 → 왼쪽 → 오른쪽`

```plaintext
현재 노드 방문  
→ 왼쪽 서브트리 순회  
→ 오른쪽 서브트리 순회
```

- 트리 구조 자체를 **복원**하거나 **복사**할 때 자주 사용.
    

#### 예시:

```
      A
     / \
    B   C
```

→ **Preorder:** `A B C`

---

### ✅ 3. **Postorder Traversal (후위 순회)**

**방문 순서:** `왼쪽 → 오른쪽 → 루트`

```plaintext
왼쪽 서브트리 순회  
→ 오른쪽 서브트리 순회  
→ 현재 노드 방문
```

- **메모리 해제**, **삭제**, **후처리 작업**에 적합.
    

#### 예시:

```
      A
     / \
    B   C
```

→ **Postorder:** `B C A`

---

### 💡 비교 요약표

|순회 방식|순서|용도|
|---|---|---|
|Inorder|왼 → 루트 → 오|BST에서 정렬된 출력|
|Preorder|루트 → 왼 → 오|트리 복원, 구조 복사|
|Postorder|왼 → 오 → 루트|트리 삭제, 후처리|

```
        A
      /   \
     B     C
    / \   / \
   D   E F   G
```

## 1. ✅ 전위 순회 (Preorder): `루트 ➝ 왼쪽 ➝ 오른쪽`

```
A ➝ B ➝ D ➝ E ➝ C ➝ F ➝ G
```

---

## 2. ✅ 중위 순회 (Inorder): `왼쪽 ➝ 루트 ➝ 오른쪽`

```
D ➝ B ➝ E ➝ A ➝ F ➝ C ➝ G
```

---

## 3. ✅ 후위 순회 (Postorder): `왼쪽 ➝ 오른쪽 ➝ 루트`

```
D ➝ E ➝ B ➝ F ➝ G ➝ C ➝ A
```

---

## 📌 순회 방식 정리표

|순회 방식|순서|
|---|---|
|Preorder|A B D E C F G|
|Inorder|D B E A F C G|
|Postorder|D E B F G C A|
