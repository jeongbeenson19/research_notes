# DSSD (Deconvolutional Single Shot Detector)

---

**DSSD**는 [[SSD]]의 한계를 개선하기 위해 제안된 객체 탐지 모델로, 특히 **작은 객체 탐지 성능**을 향상시키는 데 초점을 맞췄습니다. DSSD는 기존 SSD의 단일 특징 맵에서 발생하는 정보 손실 문제에 대응하기 위해 **역합성곱(Deconvolution)** 레이어를 도입하여 고레벨 의미론적 정보와 저레벨 특징을 융합하는 방식을 사용합니다.

## 1. 핵심 특징

### **1) Deconvolutional Layer를 통한 특징 융합**

기존 SSD는 깊은 레이어에서 나온 특징 맵일수록 해상도가 낮아져 작은 객체에 대한 공간 정보가 부족해지는 문제가 있었습니다. DSSD는 이 문제를 해결하기 위해 **Deconvolutional Layer (또는 Transposed Convolution)** 를 사용하여 특징 맵의 해상도를 높입니다.

-   **상위 레이어의 풍부한 의미론적 정보**: 깊은 네트워크 레이어의 특징 맵은 넓은 수용 영역(Receptive Field)을 가지므로 객체의 종류를 분류하는 데 유리한 고레벨 의미론적 정보를 포함합니다.
-   **하위 레이어의 정밀한 공간 정보**: 얕은 네트워크 레이어의 특징 맵은 해상도가 높아 객체의 정확한 위치 정보를 포함합니다.

DSSD는 이 두 가지 정보를 효과적으로 융합하여, **고레벨의 의미론적 정보를 저레벨의 공간 정보에 주입**함으로써 작은 객체의 분류 및 위치 예측 능력을 강화합니다. 이 융합은 종종 Element-wise Product나 Skip Connection 등의 형태로 이루어집니다.

### **2) 강력한 백본 네트워크 (ResNet-101 등)**

SSD가 주로 VGG-16을 백본으로 사용했던 것과 달리, DSSD는 ResNet-101과 같은 더 깊고 강력한 네트워크를 백본으로 채택하여 특징 추출 능력을 향상시킵니다. 이는 전체적인 탐지 성능 향상에 기여합니다.

## 2. DSSD 아키텍처

DSSD는 기본적으로 SSD와 유사한 다중 스케일 예측 구조를 따르지만, 특징 맵 생성 과정에서 Deconvolutional Module이 추가됩니다.

1.  **백본 네트워크**: ResNet-101과 같은 강력한 분류 네트워크를 사용하여 초기 특징 맵을 추출합니다.
2.  **SSD 추가 레이어**: 백본 뒤에 SSD와 유사하게 추가적인 Convolutional 레이어들을 쌓아 다양한 스케일의 특징 맵을 생성합니다.
3.  **Deconvolutional Module**: 각 스케일의 특징 맵에 Deconvolutional Module을 적용하여 해상도를 높이고, 이 과정에서 더 깊은(의미론적) 레이어의 정보를 얕은(공간적) 레이어의 정보와 융합합니다. 이를 통해 최종적으로 예측에 사용되는 특징 맵들은 더 풍부한 정보를 갖게 됩니다.
4.  **예측**: 각 특징 맵의 셀마다 디폴트 박스를 기반으로 객체의 클래스와 위치(offset)를 예측합니다.

## 3. 성능

DSSD는 기존 SSD 대비 특히 작은 객체 탐지에서 상당한 성능 향상을 보여주었습니다. PASCAL VOC 및 COCO 데이터셋에서 mAP를 크게 높였으며, Faster R-CNN과 같은 2-Stage Detector에 필적하거나 능가하는 정확도를 보이면서도 1-Stage Detector의 장점인 빠른 추론 속도를 유지합니다.

## 4. 장단점

### **장점**

-   **작은 객체 탐지 성능 향상**: Deconvolutional Layer를 통한 특징 융합으로 작은 객체 탐지에서 SSD의 약점을 보완했습니다.
-   **높은 정확도**: SSD보다 전반적으로 높은 mAP를 달성했습니다.
-   **빠른 추론 속도**: 여전히 1-Stage Detector로서 실시간에 가까운 속도를 제공합니다.

### **단점**

-   **복잡도 증가**: Deconvolutional Layer와 특징 융합 모듈의 추가로 네트워크 구조가 SSD보다 복잡해지고, 이는 학습 시간 및 메모리 사용량 증가로 이어질 수 있습니다.
-   **계산량 증가**: 추가적인 Deconvolutional 연산으로 인해 SSD 대비 추론 시간이 소폭 증가할 수 있습니다.
